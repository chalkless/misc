# venv
## 何ソレ
- プロジェクトごとに依存するPythonライブラリを切り分けた環境を作るもの（平たく言うと仮想環境）

## apt/yum/brew、conda、pyenvとの比較
- （後で書く）

## インストール（というか仮想環境の作成）
- いまどきのPython（3.3以降）だと、すぐに環境が作れるようになっている
```
python -m venv project_name
```
```
# 例
python -m venv venv
```
- システム側（aptとかyumで入れるような）をいじらずに、自分のホームディレクトリの下に環境を作れる
- 複数のプロジェクトを抱えていて、プロジェクトごとに依存するライブラリ環境を整えるならつどつどプロジェクトを作ればよい
- あくまで自分用のPythonの環境が欲しい、ということであれば、習わしとして`venv`というプロジェクトを1つ作ってそこで作業する感じらしい

## 使い方
### 使う時（アクティベート）
- インストールしただけでは動かない。（というかその環境が使えない）。アクティベートして使う
```
cd project_name
source vnev/bin/activate
```
- sourceは、設定ファイルを読み込むコマンド。`source .bashrc`というのでもおなじみ
- つまり、`project_name/bin/`に`activate`という設定ファイルがあるわけで、それを読み込んでいる
- アクティベートされると、プロンプトがそれっぽくかわる
```
$ source vnev/bin/activate
(venv) $
```

### 使い終わる
```
deactivate
```
